\paragraph{Setup.}
We stream five complete ACL talks and five medicine talks through one
alternating En$\rightarrow$Zh session (approximately 4.7 hours). All
terminology policies use 1.92s chunks, Qwen3-Omni, the same MaxSim retriever,
100 retrieval candidates, score threshold 0.78, and at most ten prompt
references. We compare:
\emph{InfiniSST (no RAG)}, historical no-retrieval outputs on the same talks at
the same chunk size \citep{ouyang2025infinisst};
\emph{Known-domain-1k}, which requires the talk domain;
\emph{AutoTerm-1k$\times$4}, which requires no session input; and
\emph{Merged-100k/1M}, two nested universal glossaries.

The ten registered AutoTerm slices contain 1k entries each. The merged
baselines combine the same target-bearing resources with distractors to exactly
100k and 1M mappings; 100k is a strict prefix of 1M. Exact duplicate pairs are
removed, but conflicting domain-dependent translations are retained. Router
settings were selected on four development windows before the complete run.
Because the registered NLP and medicine slices include benchmark terminology,
this is a controlled routing and retrieval-competition evaluation, not a
held-out glossary-induction test.

\paragraph{Comparison provenance.}
The no-RAG row reuses the public InfiniSST hypotheses released with RASST for
these exact ten talks and the matching 1.92s policy; it does not require another
model run. We resegment those hypotheses against the same references and
rescore all 2,047 term occurrences used by the RAG conditions. The other four
rows come from one audited run per policy with identical audio order and chunk
boundaries. Corpus BLEU is computed after pooling the resegmented sentences,
rather than by averaging the ACL and medicine BLEU scores.

\paragraph{Metrics.}
\textbf{TERM\_ACC} uses one fixed set of 2,047 time-aligned occurrences from
the raw ACL tagged glossary and medicine hard/raw glossary. Exact-span source
aliases with identical target variants are one spoken occurrence; nested terms
and different translations remain separate. Every condition, including no
RAG, is rescored against this same denominator.
\textbf{BLEU} follows the RASST/StreamLAAL pipeline: each talk hypothesis is
resegmented to reference sentences with mWERSegmenter before corpus
sacreBLEU. \textbf{MT-BLEU} applies the same pipeline after masking target
translations from the corresponding corpus's raw glossary. Prompt precision is
the fraction of injected references that overlap a local raw-term occurrence;
Refs is the mean number of references injected per chunk. The masking inventory
is the full raw tagged set, not a curated subset used for interface highlights;
we do not report a second filtered masked-BLEU variant.

\paragraph{From no RAG to an adaptive working set.}
Table~\ref{tab:quality} answers three systems questions. First, terminology
retrieval matters: AutoTerm improves TERM\_ACC from 51.1\% without RAG to
88.3\%, while BLEU rises by 5.83. Second, automatic routing removes setup at
negligible quality cost. AutoTerm is within 0.24 TERM\_ACC points and 0.11
BLEU/MT-BLEU of the known-domain reference, despite receiving no domain input.
Third, a universal glossary is usable at 100k but degrades as it grows. Relative
to Merged-100k, AutoTerm gains 1.37 BLEU and provides much cleaner evidence
(34.8\% vs. 6.8\% prompt precision). At 1M, its advantage grows to 3.91
TERM\_ACC points and 2.30 BLEU, while the merged baseline fills nearly the
entire top-10 prompt with low-precision references.

\paragraph{Where does the scale loss occur?}
The domain columns in Table~\ref{tab:quality} prevent the aggregate result from
hiding a useful qualification. Merged-100k is still competitive on NLP and loses only 1.8
points to AutoTerm on medicine. At 1M, however, the medicine gap expands to 7.4
points and an NLP gap also appears. Thus the result does not claim that every
large index immediately fails. It shows that, under fixed retrieval and prompt
capacity, the universal strategy becomes increasingly sensitive to the term
distribution and ambiguity of each domain; a compact routed working set avoids
that exposure.

\paragraph{Why does translation quality move?}
Prompt evidence exposes the mechanism behind the aggregate scores. AutoTerm
injects fewer than one reference per chunk on average at 34.8\% precision.
Merged-100k injects 4.96 at 6.8\%, and Merged-1M injects 8.70 at only 3.4\%.
Under the same threshold, the larger indexes fill a fixed prompt with more
low-value mappings. MT-BLEU masks the evaluated term translations, yet AutoTerm
still leads Merged-1M by 2.33 points; irrelevant evidence therefore affects
surrounding translation, not only tagged terms. Prompt precision exposes the
mechanism, while TERM\_ACC and MT-BLEU measure its consequences.

\paragraph{Routing behavior.}
Figure~\ref{fig:routing} illustrates why AutoTerm uses a multi-slice working
set rather than a hard top-1 router. Transient top-slice errors occur around
topic boundaries, yet another active slice usually preserves the correct
terminology inventory. This limits the cost of delayed topic evidence in the
generated translation context.

\paragraph{Observed failure modes.}
After one ACL-to-medicine boundary, the accumulated context retained the old
topic for several chunks, so the medicine slice was temporarily absent before
entering the four-slice working set. Lexical ambiguity creates a second error
mode. In a medicine segment discussing scientific papers, Merged-1M supplied
both the Chinese mapping for a scholarly article and an incompatible mapping
for physical paper, whereas AutoTerm retained only the context-appropriate
first sense. The example illustrates that routing reduces, rather than
eliminates, ambiguous prompt evidence.

\paragraph{Concurrent serving.}
The hosted backend uses vLLM continuous batching. In an end-to-end real-time
feed through the framework and WebSocket protocol, each client sends audio at
the 1.92s input cadence and all 32 sessions complete without failure. The test
includes scheduler admission, model generation, and WebSocket delivery; its
generation p95 remains 0.82s at 32 sessions, or 0.43 of the input stride
(Table~\ref{tab:serving}). The nearly flat p95 from 8 to 32 clients is the
expected continuous-batching behavior. We report this directly measured
service-time ratio rather than inferring an end-to-end real-time factor from
engine-only throughput.
