\begin{abstract}
We present AutoTerm-SST, an interactive system for terminology-aware streaming speech translation. Live domain talks often contain rare entities, technical terms, and rapidly changing vocabulary, yet requiring users to prepare a glossary before a session creates substantial friction. AutoTerm-SST addresses this problem with a zero-setup adaptive terminology memory: the system keeps large terminology resources offline, but uses a compact working glossary during streaming. A training-free router combines speech-side retrieval embeddings and retrieved-term metadata, without ASR or generated text, and switches to a domain slice only when confidence guards are satisfied. The demo exposes translations together with retrieved terms, active glossary state, router decisions, and retrieval latency in a live evidence panel. In a terminology smoke evaluation, the zero-setup working glossary recovers eight of nine gold terms while injecting fewer prompt references than a broad 100k open memory; separate scale probes show warm MaxSim retrieval remains within about 60--105 ms p95 up to million-term memories. AutoTerm-SST is released as a pluggable RASST-Demo framework with mock and live GPU modes.
\end{abstract}
