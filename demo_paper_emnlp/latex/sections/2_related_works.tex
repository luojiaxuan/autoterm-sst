\section{Related Work}

\paragraph{Streaming speech translation.}
SST systems must translate partial speech while balancing quality and latency. Prefix-to-prefix policies such as wait-\(k\) make this trade-off explicit \citep{ma2019stacl}, SimulEval standardizes simultaneous evaluation with quality and latency metrics \citep{ma2020simuleval}, and large-scale streaming models such as Seamless target direct simultaneous translation \citep{seamless2023}. Speech language models make this setting more flexible by combining acoustic context and multilingual generation in a single agent \citep{xu2025qwen3omni}, but live deployment still requires session management, incremental transport, and latency-aware prompting. Unlike existing streaming ST demonstrators, which fix the vocabulary context at session start, AutoTerm-SST manages terminology context \emph{during} the stream; it is a user-facing system demonstration rather than a new speech translation architecture.

\paragraph{Terminology-aware translation.}
Terminology control has long been studied in machine translation, usually through user-provided dictionaries, constrained decoding, or training-time exposure to terminology constraints \citep{hokamp2017lexically,dinu2019training}; terms and named entities are also a known weak point of speech translation specifically \citep{gaido2021moby}. Retrieval-augmented generation offers another way to expose non-parametric memory at generation time \citep{lewis2020retrieval}. These methods assume the domain knowledge is known ahead of time. AutoTerm-SST targets a different interaction pattern: users start a live speech session with no setup, and the system maintains an active terminology context from retrieval evidence observed during streaming.

\paragraph{LLM serving and demo systems.}
Modern LLM serving systems provide continuous batching and efficient generation APIs \citep{kwon2023efficient}, but a streaming speech demo also needs microphone/file input, WebSocket transport, session cleanup, live evidence rendering, and graceful mock-mode testing. AutoTerm-SST contributes this integration layer while keeping model-specific code inside swappable agents.
