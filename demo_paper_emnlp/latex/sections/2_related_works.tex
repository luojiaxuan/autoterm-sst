\section{Related Work}

\paragraph{Streaming speech translation.}
SST systems must translate partial speech while balancing quality and latency. Prefix-to-prefix policies such as wait-\(k\) make this trade-off explicit \citep{ma2019stacl}, and SimulEval standardizes simultaneous text and speech evaluation with quality and latency metrics \citep{ma2020simuleval}. Speech language models make this setting more flexible by combining acoustic context and multilingual generation in a single agent \citep{xu2025qwen3omni}, but live deployment still requires session management, incremental transport, and latency-aware prompting. AutoTerm-SST builds on this line of work as a user-facing system demonstration rather than proposing a new speech translation architecture.

\paragraph{Terminology-aware translation.}
Terminology control has long been studied in machine translation, usually through user-provided dictionaries, constrained decoding, or training-time exposure to terminology constraints \citep{hokamp2017lexically,dinu2019training}. Retrieval-augmented generation offers another way to expose non-parametric memory at generation time \citep{lewis2020retrieval}. These methods are valuable when the domain knowledge is known ahead of time. AutoTerm-SST targets a different interaction pattern: users start a live speech session with no setup, and the system maintains an active terminology context from retrieval evidence observed during streaming.

\paragraph{LLM serving and demo systems.}
Modern LLM serving systems provide continuous batching and efficient generation APIs \citep{kwon2023efficient}, but a streaming speech demo also needs microphone/file input, WebSocket transport, session cleanup, live evidence rendering, and graceful mock-mode testing. AutoTerm-SST contributes this integration layer while keeping model-specific code inside swappable agents.
