\section{Router Configuration and Guards}
\label{app:router}

Table~\ref{tab:router-config} lists the default production configuration of
the hybrid active-glossary router as released in
\texttt{configs/autoterm\_slices.yaml}; environment variables can override
individual values per deployment. Guards are evaluated in order: warmup,
update interval, target-equals-active, index readiness, probe floors,
confidence, top-vs-runner-up margin, target-vs-current margin, post-switch
cooldown, and consecutive-window consistency. Uncertain windows keep the
current slice. The 640-window state-machine benchmark in
Section~\ref{sec:evaluation} ran under the stricter three-window
generated-target consistency setting; the released default was subsequently
retuned to two windows with a raised target-vs-current margin (0.30) based on
observed real-probe failure modes.

\section{Glossary-Scale Streaming Sweep}
\label{app:sweep}

Table~\ref{tab:sweep} reports the five-talk streaming sweep referenced in
Section~\ref{sec:evaluation}: five concatenated ACL 60-60 talks (3{,}442s of
audio) streamed through the live JSON WebSocket at latency multiplier 2, scored
against a curated 142-term technical gold set. Each inventory always contains
the full 238-entry curated ACL glossary; larger rows add Wikidata-mined AI/CS
candidates as distractors. These are manual-preset runs isolating inventory
scale (not \texttt{auto\_working} runs). Output term accuracy, BLEU, and
masked BLEU stay within a narrow band up to 100k terms, while retrieval
precision collapses --- the prompt channel fills with distractors long before
output quality visibly degrades, motivating compact routed active slices.
